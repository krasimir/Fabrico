Installation

https://github.com/krasimir/Fabrico

Run fabrico

http://[path to fabrico's files]/admin
http://[path to fabrico's files]/docs
http://[path to fabrico's files]/custom
http://[path to fabrico's files]/console
http://[path to fabrico's files]/...

Units

What is unit?

The purpose of the unit is to hold specific logic, which means that it is a good idea to split your application logically and create as many units as you need. The default units are:

  • admin - solution for administrating models
  • custom - an example of unit
  • console - console unit
  • docs - the documentation of Fabrico

Structure of a unit

Every unit is placed in its own directory. To create a unit you need two files

  • .htaccess
    • RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^(.*)$ index.php?slug=$1 [QSA,L]
  • index.php
    • <?php

    require("../global.php");
    
    $fabrico = new Fabrico(
        array(
            // describe the unit's modules here
        )
    );

?>
Please check the custom unit. It's role is to give you a rough idea of how to create a unit.

Access

Restricts the access to the unit.

Configuration

{
    "type": "access",
    "title": "Login",
    "users": [
        {
            "username": "<string>",
            "password": "<string>",
            "type": "<string>",
            "<your custom property name here>: "<your custom property value here>"
        }
    ]
}

Example

{
    "type": "access",
    "users": [
        {
            "username": "admin",
            "password": "admin"
        }
    ]
}

Adapters

Holds the settings for the database connection. Required if you are going to use Models.

Configuration

{
    "type": "adapters",
    "MySQL": {
        "host": "<host>",
        "user": "<user>",
        "pass": "<pass>",
        "dbname": "<database name>"
    }
}

Example

{
    "type": "adapters",
    "MySQL": {
        "host": "localhost",
        "user": "root",
        "pass": "",
        "dbname": "fabrico_test"
    }
}

Assets

Compilation and including of css and javascript.

Configuration

{
    "type": "assets",
    "assets": [
        <asset>,
        <asset>,
        <asset>,
        ...
    ]
}
Asset
{
    "name": "<name of the asset>", 
    "sourceToCompile": [
        <path to file>,
        <path to file>,
        <path to file>,
        ...
    ],
    "sourceToInclude": [
        <path to file>,
        <path to file>,
        <path to file>,
        ...
    ],
    "destination": "<path to dir>",
    "extension": "<js, css, ...>",
    "preventCompiling": <boolean>
}

Example

{
    "type": "assets",
    "assets": [
        {
            "name": "javascript", 
            "sourceToCompile": [
                "/js/jquery/jquery-1.7.1.min.js",
                "/js/calendar/*.js",

                "/js/utils/*.js",
                "/js/presenters/*.js",

                "/js/tinymce/tiny_mce.js"
            ],
            "destination": "/assets/compiled/",
            "extension": "js",
            "preventCompiling": false
        },
        {
            "name": "css", 
            "sourceToCompile": [
                "/assets/css/*.css"
            ],
            "sourceToInclude": [
            
            ],
            "destination": "/assets/compiled/",
            "extension": "css"
        }
    ]

Router

Setup the routs of your unit here.

Configuration

{
    "type": "router",
    "routes": [
        <Route>,
        <Route>,
        <Route>,
        ...
    ]
}
Route
{
    "url": "<regular expression>", 
    "controller": {
        "class": "<path to controller>",
        "uid": <used only if you are working with a controller for the admin unit>
    }
    "action": "<method of the controller (optional)>", 
    "priority": <boolean (default: false)>
}

Example

{
    "type":"Router",
    "routes":[
        {
            "url":"/testcontroller(.*)?",
            "controller":
            {
                "class":"controllers/Controller.php",
                "uid":"testcontroller"
            },
            "action":"run",
            "priority":false
        },
        {
            "url":"(.*)?",
            "controller":
            {
                "class":"controllers/Home.php",
                "config":null
            },
            "action":"run",
            "priority":false
        }
    ]
}

    "field": "<name of field>",
    "shouldMatch": "<regular expression>"
}


// A complex logic could be added in the dependencies' section. For example:
"dependencies": [
    a,
    b
]
// means: a && b. While:
"dependencies": [
    a,
    [
        b
    ]
]
// means: a || b.
// note that
"dependencies": [
    [
        b
    ],
    a
]
// still means a && b.
// So, when Fabrico reaches an array it uses ||, otherwise &&. Few examples:
"dependencies": [
    a,
    [
        b,
        c
    ]
]
// -> a || (b && c)
"dependencies": [
    a,
    [
        [
            b,
            c
        ],
        [
            d,
            e
        ]        
    ]
]
// -> a || ((b && c) || (d && e))
"dependencies": [
    [
        a
    ],
    [
        b     
    ]
]
// -> a || b
"dependencies": [
    [
        b,
        [
            c
        ]
    ],
    a
]
-> a && (b || c)

{
    "class": "<path to validator php>",

    },
